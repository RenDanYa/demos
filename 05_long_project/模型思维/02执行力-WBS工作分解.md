### **配套工具包**
#### 1. WBS分解逻辑图
```mermaid
graph TD
    A[项目总体] --> B[需求分析]
    A --> C[系统设计]
    A --> D[开发实施]
    B --> B1[用户访谈]
    B --> B2[需求文档]
    C --> C1[架构设计]
    C --> C2[数据库设计]
```

#### 2. 分解原则（**100%规则**）
> 上级任务范围 = 所有下级任务范围之和（避免遗漏或溢出）
> **正确示范**：
> `1.0 网站开发（100%）` = `1.1 前端(40%)` + `1.2 后端(35%)` + `1.3 测试(25%)`

#### 3. 风险标识规则
| 风险等级 | 判定标准                          | 应对措施                  |
|----------|----------------------------------|--------------------------|
| 高       | 影响进度＞3天／成本超支＞20%      | 制定预案／每周专项跟进    |
| 中       | 影响进度1-3天／成本超支5-20%      | 双周检查／储备资源        |
| 低       | 影响＜1天／成本偏差＜5%           | 月度监控                  |

### **使用流程**
1. **步骤1：定义交付物**
   - 在表格“工作包/交付物”列填写**名词型成果**（如“测试报告”而非“进行测试”）
 
2. **步骤2：层级分解**
   - 按 **`可管理/可分配/可估算`** 原则拆分（单任务工时建议≤40h）
   - 使用WBS编码确保唯一性（推荐工具：Project、Excel树状图）

3. **步骤3：动态跟踪**
   - **状态标记**：□未开始 ◉进行中 ✓完成 ⚠延期
   - **每周更新**：根据进度刷新“状态跟踪”列

> 💡 **避坑指南**：
> - ✘ 避免按部门分解 → 应**按交付成果**分解
> - ✘ 任务粒度＞5天 → 需继续拆解
> - ✔ 所有叶子节点（最末级）必须有**可量化交付物**
