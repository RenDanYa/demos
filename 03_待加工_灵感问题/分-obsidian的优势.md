# 2025-09-20
- 拆分笔记的好处
	- 问ai问题的时候可以只针对一个问题，深入，直到当下感觉问题被解决
	- 之前写了6,7页去问ai
		- 一是结论不是自己得出来的，总是无法适配我，不是我想要的答案，然后归咎于指令调不对
		- 二是给出行动项又真的很难去执行，
	- 更好的知道ai扮演的角色了，专攻，而非将问题一整个甩给ai，然后又没有办法拆分、多轮问答，问题不明确，答案一大块，直接执行又不是从内驱动力上进行了自我说服，没有办法发挥Ai的效力
- 而且双链和我之前认知到的也不一样，我之前一直以为手动，我心想那不死掉了，纯粹是为了联结而联结，还得我自己找他们之间的关系，那现在这种联结就是自然而然的效果，是工具在帮我进行联结
- 还有一个好处就是他允许我的思想以半成品的方式存储，如果写到notion中对我来说压力太大了，他的结构化使得我很有压力，好像要马上处理未被归类的笔记，不允许有孤立笔记的存在，现在在obsidian中我把主题归类了，但解决方案很多还没有找到，但我允许他以这种形态存在

# 2025-11-15
- 我觉得Obsidian是我很多东西的载体
	- 是我时间记录的载体
		- 目前我这个功能用下来，几乎把之前的起床，休息这些都去掉了，只留下核心标签，比如工作，播客这些
			- 因为休息首先是标签细，但是我又不对他做记录，导致他其实对我整体的时间调节没有太大的作用
			- 或者说他整体的占比时间不长，多几十分钟或者少几十分钟，我都不一定在意，因为有其合理性，我只关心工作+播客+阅读+运动时间
		- 还有一部分是记录本身也繁琐，那我抓大放小，也是没有问题的
			- 那我想记的时候记录没有问题，不想记录的时候那我就不再去强求了
	- 是我习惯记录的载体
	- 是我个人思考的载体
		- 我知道我对我失败的感悟还不够，但Obsidian并没有让这件事情画上句号，而是让我每接触一个新的概念，就把之前的东拿出来剖析，让我有了成长性的心态，这也算这个概念的实践性工具吧，对我来说不是一个抽象的概念，而是具体可实施可落地的方案，虽然没有人知道方案是什么，但我知道总有一天他会浮现的
	- 还是我工作项目资料整理的载体
	- 他真的何尝不是我的第二大脑，我觉得当之无愧吧，我的大脑只能捕捉一个灵感，但是他让我有了探索下去，寻找到解决方案的可能
- 这个仪表盘
	- 首先让我微启动了很多事情，这是不可否认的
		- 比如阅读，播客，运动
		- 很多事情的长期主义，在我这里算是埋下了种子
	- 接下来就是执行过程中的调整了
		- 我知道很多事情都是迭代的，是螺旋上升的，所以出现问题我也不怕，无法就是遇到一个问题，解决一个问题，直到行为彻底固化下来
